Create a configuration

Configurations are hyperparameters that the trainer uses during training. These are individual for every classifier. There are some configuration options that are the same for all classifiers. These are:

  • Folds: The trainer uses n-fold cross-validation. This means, that it divides the known documents into n subsets of roughly equal size. The machine learning system is then trained n times. Each time, it learns from n-1 subsets and is evaluated with the remaining one. The subset that worked best (see selection policy), is used for the active model. This parameter defines how many such training/evaluation sessions take place. They are run in parallel. The default parameter is 3.
  • Include Implicits: If the categories have relationships between each other, documents can implicitly belong to others. For example, if there are the categories cats, and mammals with cats being a sub-category of mammals, a document that belongs to cats implicitly also belongs to mammals. This parameter controls if these implicit assignments are taken into account during training and evaluation. The default parameter is true.
  • Assignment Threshold: Certain classifier algorithms generate probabilities of documents belonging to certain categories. In order to evaluate which fold worked best, this parameter controls the minimum probability such an assignment needs to have in order to be considered as belonging to a certain document. High values boost precision, while low values boost recall (see selection policy). The default parameter is 0.5, balancing both measures.
  • Selection Policy: This parameter defines the selection policy during n-fold cross-validation. Precision means, that the amount of documents wrongly assigned to a category is minimized. Recall means, that the amount of documents missing from a category is minimized. F1 is a equal weighted combination of both. Microaverage measures emphasis overall performance, while macroaverage measures emphasise getting "small" categories with little belonging documents right. The default parameter is Microaverage F1, emphasis overall performance while balancing precision and recall.
  • Top Terms per Category: This parameter defines how many terms are added to the controlled vocabulary. The decision to add a document to the controlled vocabulary is based on the terms with the top tfidf values for documents in the category.
  • SVM type: The underlying classifier implements the LibSVM library (official website). In regards of text classification, two of the multiple SVM modes supported by the library can be used. SVM stands for Support Vector Machine. https://en.wikipedia.org/wiki/Support_vector_machine. It uses hyperplanes to separate data points into belonging to a certain category or not. It allows for exception points that can be on the wrong side of the hyperplane. C, and nu are regularisation parameters. They are used to implement the penalty for misclassifications during leraning. The penalty for c_svm can be between 0 and infinity. The penalty for nu_svm can be between 0 and 1. The default selection is c_svm
  • Kernel type, degree, gamma and coef0: SVMs use kernels. These are mathematical formulas to map existing vectors into higher-dimensional space. This is done to easier separate datapoints from each other. The values for degree, gamma and coef0 are used in the kernel formulae. Their default values are:
    • Degree: 3
    • Gamma: 1/k. Here k is the dimensionality of data points. For word embeddings it's the dimensions of the used word embedding
    • coef0: Default is 0.
    • SIGMOID: tanh(gamma*u'*v + coef0)
    The formuale are:
    • Linear: u'*v
    • Poly: (gamma*u'*v + coef0)^degree
    • RBF: exp(-gamma*|u-v|^2) This is the default!
    • SIGMOID: tanh(gamma*u'*v + coef0)
